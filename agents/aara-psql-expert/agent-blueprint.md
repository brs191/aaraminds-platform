## Business Problem

Teams write, review, and optimize PostgreSQL stored procedures (PL/pgSQL), functions, triggers, and complex queries by hand, with inconsistent quality, no shared review discipline, and frequent correctness and performance defects. Expertise is uneven and schema knowledge is often assumed rather than verified, which produces subtly wrong SQL that fails only in production.

## Expected Outcomes

- Reviewed, evidence-grounded PL/pgSQL and SQL drafts produced from provided schema
- Every schema or behavior claim cited to the provided DDL, never assumed
- Performance and correctness issues flagged with a rationale before code is run
