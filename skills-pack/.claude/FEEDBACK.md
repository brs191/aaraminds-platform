# Pack-usage feedback log

Chronological log of observations from using this pack in real work. Claude reads this at the start of pack-related sessions and adjusts behavior to reflect what's here. The user appends entries at the end of meaningful sessions (or whenever something is worth capturing).

## Format per entry

```
## <ISO date> â€” <one-line context>
- What worked: <observation>
- What failed: <observation, ideally specific to a skill / agent / hook>
- Action: <proposed edit to a SKILL.md, agent, hook, or "leave alone, one-off">
```

Entries stay until the quarterly synthesis pass (see `../validation/governance/freshness-cadence.md`), at which point patterns are folded into SKILL.md / agent / hook edits and absorbed entries are archived or truncated.

Narrow per-skill feedback can also land in `skills/<name>/references/notes.md` instead of here. Default catch-all is this file.
